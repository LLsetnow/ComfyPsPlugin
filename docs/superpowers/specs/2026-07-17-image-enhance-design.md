# 图像高清与云扉工作流扩展设计

## 目标

新增“图像高清”工作流，并让云扉原生 ComfyUI 后端支持“图像高清”“背景去杂物”和“面部重绘”。每个工作流继续通过本地桥执行，成功结果作为 Photoshop 新图层贴回。

## 范围与约束

- “图像高清”提供“图像清晰”和“图像放大”两种模式。
- 两种模式均为单图输入、单图输出，不需要选区或蒙版；输入为活动文档的完整画布。
- 两种模式提供 `1` 到 `8` 的浮点“放大比例”，默认值为 `2`，写入工作流节点 `95.inputs.value`。
- “图像清晰”使用 RunningHub 工作流 `2078092574119964674`，并保持原始分辨率；“图像放大”使用 `2078099177921589250`，并输出放大后的分辨率。
- 两种 JSON 工作流从 `RH_CLI/workflows/ai/` 纳入仓库，命名为 `workflows/image_clarity_api.json` 和 `workflows/image_upscale_api.json`。
- RunningHub、本地 ComfyUI、云扉 ComfyUI 都必须支持该工作流。
- 云扉还必须支持现有“背景去杂物”和“面部重绘”；已有 Boogu 局部编辑行为不改变。
- 插件 JavaScript 必须维持 ES5/UXP 兼容。

## 工作流元数据

`plugin/main.js` 中的工作流定义成为各后端共享的运行契约：

- 现有工作流新增云扉执行所需的输出节点与提示词目标元数据：背景去杂物使用输入节点 `41`、输出节点 `220`、提示词 `68.prompt`；面部重绘使用 `27`、`72`、`2.text`；Boogu 局部编辑继续使用其既有专用节点。
- 新的 `image-enhance` 工作流通过模式值选择对应的 `workflowId`、仓库内 JSON 路径、输入节点 `90` 和输出节点 `100`。
- 新工作流的 `setArgs` 将已验证的比例序列化为 `95:value=<number>`。前端会将该参数传给所有三个后端。
- `getWorkflowRunConfig()` 会把模式专属配置传递到桥；`isWorkflowAvailableForBackend()` 会允许云扉执行这三个单图工作流，而仍禁用其他未声明为云扉原生兼容的工作流。

## 桥接与数据流

前端会将图像、工作流路径、输入/输出节点、可选蒙版、提示词及受控的 `extraSetArgs` 提交到 `/run`。

1. RunningHub 保持现有 `run_inpaint_blocking()` 调用模型，使用模式对应的 workflow ID，并注入 `95:value`。
2. 本地 ComfyUI 执行器扩展为读取请求指定的工作流文件与节点：注入图片，可选注入蒙版和提示词，应用受控参数，然后从请求指定的 SaveImage 节点读取结果。这样“图像高清”的单图路径不会尝试读取或上传蒙版。
3. 云扉适配层新增通用原生 ComfyUI 执行函数：发现运行实例、上传原图（以及仅在需要时上传蒙版）、深拷贝工作流、按元数据注入图片/蒙版/提示词/参数/文件名前缀、轮询指定输出节点并下载 PNG。它不向实例的 ComfyUI 接口发送云扉 Token。
4. 现有 `run_native_inpaint()` 与其 Boogu 节点替换逻辑保留为兼容封装；桥根据工作流元数据仅对 Boogu 局部编辑使用它。其它三个云扉工作流走通用单图路径。

桥仅接受仓库 `workflows/` 内的相对工作流路径；缺少节点、非法参数格式、缺少输出或云扉实例没有所需节点/模型时，返回不含凭证的明确错误。

## 用户体验与错误处理

- “图像高清”卡片显示模式选择和比例滑杆；描述明确区分“清晰不改变分辨率”与“放大改变分辨率”。
- 云扉后端下，“图像高清”“背景去杂物”和“面部重绘”卡片可点击；其余未兼容工作流仍显示为不可用。
- 所有无蒙版工作流导出整个活动文档，并在返回后创建 `ComfyPS - <工作流名>` 图层。
- 比例在前端和桥接层都限制为有限数值 `1–8`；非法值不会传给后端。

## 测试策略

- Node 测试验证新工作流的两种模式会产生正确的 RunningHub ID、JSON 路径、节点 `90`/`100` 和 `95:value`，并验证云扉可用性只扩展到指定工作流。
- Python 单元测试验证通用云扉执行路径：不带 Authorization 的 ComfyUI 请求、单图上传、提示词和比例注入、从配置输出节点下载结果；并覆盖可选蒙版路径的兼容性。
- Python 单元测试验证 `/run` 为本地 ComfyUI 与云扉转发请求工作流元数据和参数，不再以 `needsMask=false` 拒绝上述云扉工作流。
- 运行 Python 语法检查、现有 Node 测试、桥接测试和 workflow JSON 校验；最后运行 UXP ES5 禁用语法扫描。

## 非目标

- 不创建或配置云扉实例镜像，也不在插件内安装 SeedVR2、面部重绘或去杂物所需的节点与模型；实例缺少依赖时由 ComfyUI 返回执行错误。
- 不把其他现有工作流自动声明为云扉兼容。
