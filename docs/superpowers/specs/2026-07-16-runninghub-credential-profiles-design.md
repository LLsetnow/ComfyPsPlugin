# RunningHub 多凭据与站点余额设计

## 目标

让 ComfyPS 保存任意数量的 RunningHub API Key，并把每个 Key 固定绑定到 `ai` 或 `cn` 站点。用户可在设置页按站点分组查看凭据、RH 币、余额和并发能力；点击一张有效凭据卡片后，后续任务立即使用该凭据及其站点。

本功能不限制凭据数量，也不允许将一个已保存 Key 复用于两个站点。

## 设置页交互

设置页中的“RunningHub 凭据”替代现有单个 API Key 输入框，使用已确认的方案 C：

- 凭据按 `ai`、`cn` 两个站点分组，组内按创建顺序排列。
- 每张卡片显示：自定义名称、站点、掩码 Key、RH 币、金额余额、检测时间和并发能力。
- 点击一张有效卡片立即将其设为当前凭据；无保存按钮。卡片和底部摘要同步标记“当前使用”。
- 当前凭据摘要明确显示“支持并发”或“单任务”。
- “添加凭据”始终位于列表底部，打开名称、固定站点和 Key 的表单。保存时立即执行检测；检测失败的凭据保留并标记错误，但不能激活。
- 编辑允许修改名称或替换 Key；站点在创建后固定。若需换站点，创建另一张凭据。
- “刷新余额”按顺序检测全部已保存凭据；切换卡片时先立即显示该卡的缓存余额，并后台刷新该卡。

参考交互稿位于 `docs/prototypes/runninghub-key-profiles-reference.html`。

## 数据模型与持久化

使用一个新的 localStorage JSON 数组保存凭据，例如 `comfyps.rhCredentials`：

```js
{
  id: "rh_...",
  name: "AI 主账户",
  site: "ai",                 // 创建后不可改
  apiKey: "rh-...",
  coins: "628",
  balance: "3.42",
  symbol: "$",
  apiType: "SHARED",
  supportsParallel: true,
  status: "ready",            // ready | error | unchecked
  errorMessage: "",
  checkedAt: 0
}
```

当前选择单独存为 `comfyps.activeRhCredentialId`。实际运行、余额显示和并发判断都从激活的凭据派生，不再依赖全局单值 `rhSite`、`apiKey` 或 `apiType`。

首次加载时迁移旧值：如果新数组为空且原有 `comfyps.apiKey` 不为空，创建一条凭据，继承旧 `rhSite` 和 `apiType`，初始状态为 `unchecked` 并选为当前凭据。设置页首次呈现时后台检测它；检测成功前不能提交 RunningHub 任务。旧键保留为兼容回退，不再作为正常运行来源。

## 数据流与并发

1. 设置页加载本地凭据和当前凭据 ID，按站点渲染卡片。
2. 新增、编辑、切换或刷新时，插件调用现有 `/test-key`，并把响应中的余额、RH 币和 `api_type` 写回对应凭据。
3. 以 `api_type` 推导 `supportsParallel`：共享/企业级为支持并发，其他类型为单任务。
4. 任务启动前只读取当前凭据，将其 `site` 与 `apiKey` 放入现有 `/run` 请求。
5. 并发调度只使用当前凭据的 `supportsParallel`。切换凭据不会中断已提交任务；每个任务在启动时保留当时的请求参数。

桥接层继续只接收单个请求中的 `site` 和 `apiKey`，无需存储或管理凭据列表。

## 错误与边界情况

- 没有凭据、当前凭据未检测或当前凭据无效时，阻止 RunningHub 任务并提示新增或重新检测凭据。
- 刷新失败不清除上一次成功余额；当前凭据会保留选中状态但不可提交，直到下次检测成功。卡片显示检测错误和旧数据的检测时间。
- 删除当前凭据后，自动选择下一张 `ready` 凭据；若不存在，进入未选择状态。
- 删除或编辑不会影响正在运行的任务。
- 无上限列表沿用设置页滚动容器，避免挤压其他设置项。
- Key 始终使用现有 localStorage 存储方式，界面只显示掩码，桥日志不得打印 Key。

## 验证

- 单元/纯函数测试：旧设置迁移、按站点分组、激活切换、删除当前凭据、并发能力推导。
- 开发桥模拟不同站点、余额和 API 类型；刷新多项时确保失败项不影响后续项。
- 手动验证：新增任意数量的 ai/cn 凭据、直接点击切换、任务请求使用选中凭据、并发状态随卡片切换更新。
- 保持 Python 编译、工作流 JSON 和 UXP ES5 兼容检查通过。
