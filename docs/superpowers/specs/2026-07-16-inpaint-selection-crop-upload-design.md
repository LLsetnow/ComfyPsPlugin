# 局部编辑选区裁切上传设计

## 目标

缩小 RunningHub 局部编辑的上传体积。QwenImage 与 Boogu 不再上传整张文档和整画布蒙版，而是上传活动图层在当前选区外接矩形内的图像，以及同一矩形裁切后的选区蒙版。

## 范围

仅修改需要蒙版的 RunningHub「局部编辑」执行路径。QwenImage 和 Boogu 共用该路径；背景去杂物、面部重绘、GPT Image 及本地 ComfyUI 路径保持不变。

## 数据流

1. 复用 `exportActiveLayerSelectionPNG()` 导出活动图层的选区外接矩形，并取得经过整数化的原始坐标与尺寸。
2. 调用 `exportSelectionMaskPNG(false, true, bounds)`，以相同矩形导出白色代表编辑区的裁切蒙版，并保留原选区快照通道。
3. 将裁切图片、裁切蒙版、提示词和当前工作流路由配置提交给本地桥。桥及 QwenImage/Boogu 工作流继续接收一个图片和一个蒙版，不修改其 API 格式。
4. 将 RunningHub 返回的裁切结果保存到队列，并保存 `bounds` 作为 `placement`。
5. 导入或即时贴回时，使用 `placement` 将结果缩放并定位到原文档坐标；再使用保留的选区快照创建同位置的图层蒙版。

## 边界与错误处理

- 沿用现有选区边界向外取整及画布边界限制，确保图像和蒙版尺寸完全一致，避免一像素偏移。
- 无选区、无活动文档或导出失败继续使用已有错误提示与清理逻辑。
- 局部编辑的本地蒙版调试模式保留整画布图片/蒙版导出与贴回逻辑，不走裁切路径，保证调试结果不变。
- 普通局部编辑必须有选区；活动图层被裁切导出时，其他图层保持隐藏，行为与 GPT Image 的活动图层编辑一致。

## 验证

- 对 QwenImage 与 Boogu 分别断言请求的图片、蒙版共享相同的裁切尺寸，且带有原始 `placement` 坐标。
- 验证返图通过即时贴回和队列导入都会按照 `placement` 定位，并创建选区蒙版。
- 验证本地蒙版调试保持整画布输出。
- 执行 Python 语法检查、工作流 JSON 校验、UXP ES5 扫描，并同步 `plugin/main.js` 到 Photoshop UXP 插件目录。
