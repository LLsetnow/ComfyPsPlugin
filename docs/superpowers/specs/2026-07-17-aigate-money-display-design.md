# 云扉金额展示设计

## 目标

云扉账户余额和 GPU 规格价格由接口以“分”为单位返回。插件设置页必须以人民币元显示，避免把 `205` 误读为 205 元。

## 展示规则

- 余额 `205` 显示为 `¥ 2.05`。
- GPU 价格 `199` 显示为 `¥ 1.99`。
- 有效整数金额始终保留两位小数；零值显示为 `¥ 0.00`。
- 空值、非数字或负数显示为“金额暂不可用”，不进行猜测性转换。

## 边界与数据流

bridge 继续把云扉 API 返回的 `balance` 和 SKU `price` 原样传给面板。`plugin/settings.js` 新增纯展示格式化函数，由余额卡、GPU 选择行和创建确认页共同使用；创建请求仍只提交 `skuName`，不传金额。

## 验证

前端单元测试覆盖余额和价格的分转元、零值与无效值。README 更新为说明余额和 GPU 价格在界面中按人民币元展示，但 bridge 响应仍保持原始数值。
