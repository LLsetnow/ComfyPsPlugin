# ComfyUI 连通性测试设计

## 目标

在设置页的本地 ComfyUI 配置中，让用户主动确认当前地址是否可供 ComfyPS 的本地桥访问。

该测试只验证服务可达；不检查节点、模型或工作流，也不创建、提交或取消任何 ComfyUI 任务。

## 方案

面板不直接请求 ComfyUI，而是调用本地桥新增的 `POST /test-comfyui`。桥使用与实际本地工作流相同的网络环境，请求用户提供地址的 `GET /system_stats`。

选择桥代测而非面板直连，原因是它避免 UXP/浏览器跨域差异，并验证真实运行路径（bridge → ComfyUI），而非仅验证 Photoshop 面板能否发起 HTTP 请求。

## 设置页交互

- 在“ComfyUI 地址”输入框同一行增加“测试连接”按钮。
- 点击时读取当前输入值；不要求用户先失焦或保存。
- 测试期间按钮禁用并显示“检测中…”。
- 成功时，在地址下方显示“已连接”；若响应包含版本字段，则附带 ComfyUI 版本。
- 失败时，显示可操作的错误信息：地址为空、桥不可达、请求超时、HTTP 状态码或连接失败原因。
- 测试结果只保留在当前设置页会话中；地址仍沿用既有自动保存逻辑。

## 桥接口

### `POST /test-comfyui`

请求体：

```json
{ "comfyuiUrl": "http://127.0.0.1:8188" }
```

桥将地址去除末尾斜杠后请求 `GET /system_stats`，超时时间为 5 秒。

成功响应：

```json
{ "ok": true, "status": 200, "version": "..." }
```

失败响应：

```json
{ "ok": false, "status": 0, "message": "..." }
```

服务可达的判定为：请求返回 2xx，且响应体是 JSON 对象。版本字段仅用于展示，不影响成功判定。桥不会将用户输入的 URL 或响应正文写入持久化配置。

## 开发预览与验证

- 开发服务器实现对应 mock 端点，返回成功和演示版本信息。
- 覆盖成功、空地址、桥不可用/超时和非 2xx 响应的面板状态。
- 运行现有 Python 语法检查和 UXP ES5 兼容性检查。

## 范围边界

不新增周期性检测、不读取 `/object_info`、不校验模型/自定义节点，也不改动本地 ComfyUI 的运行协议。
