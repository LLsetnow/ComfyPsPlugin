# 面板加载时强制替换本地桥设计

## 目标

每次 ComfyPS 面板加载时，强制结束旧的 ComfyPS `bridge.py` 进程并启动新的桥进程，而不是仅在健康检查失败后启动。

## 触发与界面

面板初始化时执行一次强制启动请求。该请求不读取桥健康状态，也不读取已删除的“桥离线时自动启动”设置。用户原有的手动“重启桥”按钮和 `/restart` 端点保持不变。

删除设置页中“打开插件时自动启动桥（离线时）”复选框，以及对应的 localStorage 键、渲染和保存逻辑。

## 启动替换逻辑

`plugin/start_bridge.command` 在启动新的 Python 进程前检查 TCP 8765 的监听进程。它只终止命令行包含仓库内 `bridge/bridge.py` 的 ComfyPS 桥进程，然后等待端口释放并执行新的 `bridge.py`。

若 8765 被其他程序占用，启动脚本退出并显示端口冲突；不会杀掉开发预览或任意无关服务。脚本仍优先使用仓库 `.venv` 的 Python，其次使用系统 Python。

## 面板状态与错误处理

面板复用现有 `startBridgeViaShell()` 的 UXP shell 授权、状态栏、日志和健康轮询。初始化触发拥有独立的一次性保护，避免同一次面板加载中重复打开终端。启动脚本失败或桥在轮询期限内未就绪时，现有错误状态继续显示。

## 验证

- Node 测试确认初始化调用强制启动路径，且不再引用旧设置。
- 启动脚本静态测试确认只终止 `bridge.py` 进程，并会拒绝非桥进程占用的端口。
- 运行现有前端、Python 桥接回归、Python 编译与 UXP ES5 检查。
