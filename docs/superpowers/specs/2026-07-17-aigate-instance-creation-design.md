# 云扉 ComfyUI：创建实例

## 目标

在 ComfyPS 的“设置 → 后端 → 云扉 ComfyUI”中，让用户能在云扉控制台不存在任何实例时创建一台预设 ComfyUI 实例；用户可比较可用 GPU 的实时价格，并能在同一连接器内查看云扉账户余额。

本功能延续现有本地桥接模型：UXP 面板只将 Bearer Token 传给本地 bridge，bridge 才访问云扉 OpenAPI。Token 不写入日志、实例生命周期记录或接口响应。

## 已确认的体验

### 云扉账户连接器

云扉设置区在 Bearer Token 下新增“云扉账户”状态卡，包含：

- 连接状态；
- 当前账户余额；
- 余额专属刷新按钮；
- 余额方块内部的最后更新时间。

余额读取失败时，余额方块改为“余额暂不可用 / 重试”，但不会禁止实例列表刷新或创建流程。余额只在账户连接器中展示，避免与 GPU 价格混淆。

### 创建入口

“创建一个 ComfyUI 实例”是空实例状态的唯一内容：

- 云扉实例列表为空时，显示创建卡；
- 控制台有任意实例时（创建中、运行中、已停止或其他未释放状态），隐藏整个创建卡及所有新建按钮；
- 创建请求提交后，立即切换为“创建中”的实例行，防止重复创建；
- 实例列表刷新失败时不基于旧列表推断为空，因而不显示创建卡。

### GPU 和价格

创建卡显示云扉实时可用 SKU 的单选列表。每行从左到右展示：选中状态、SKU/GPU 名称、显存或规格信息、价格。价格在确认页再次展示，格式由 bridge 生成的 `priceLabel` 提供；前端不自行猜测云扉 `price` 字段的单位或换算规则。

价格无法读取时，该规格显示“价格暂不可用”，且“继续”不可用；用户可刷新 SKU。这样不会用过期或含义不明的价格开始计费。

### 创建流程

1. 用户选择一项可用 SKU；
2. 点击“继续”，创建卡原位切换为确认摘要；
3. 摘要显示 SKU、GPU/规格、实时价格、预设 ComfyUI 镜像，以及“确认后开始按云扉最终价格计费”的提醒；
4. 点击“确认创建”后，原位显示真实生命周期状态（已提交、分配算力、等待 ComfyUI 就绪），不伪造百分比；
5. bridge 返回或后续刷新发现实例后，显示现有实例列表和既有的启动/关闭/释放动作。

## 前端组件和状态

所有 UXP JavaScript 保持 ES5 兼容；DOM 由 `plugin/settings.js` 创建，样式放入 `plugin/styles.css`，静态结构锚点位于 `plugin/index.html`。

| 状态 | 可见内容 | 可用操作 |
| --- | --- | --- |
| 未输入 Token | Token 提示 | 输入 Token、刷新 |
| 正在读取 | 骨架/加载提示 | 无重复请求 |
| 实例为空 | 账户余额卡、GPU 创建卡 | 刷新余额/规格、选择 SKU、继续 |
| 确认创建 | 配置及价格摘要 | 返回、确认创建 |
| 创建中 | 生命周期步骤、自动刷新实例 | 无重复创建 |
| 已有实例 | 账户余额卡、实例列表 | 刷新、启动、关闭、释放 |
| 请求失败 | 对应区块错误和重试 | 重试；不依据不完整结果显示创建卡 |

## Bridge API

在现有 `/aigate/*` 接口旁增加最小专用端点。所有端点接收 `aigateToken`；bridge 统一标准化 Bearer 前缀，并将云扉错误转换为无凭证泄露的用户消息。

| 面板端点 | bridge 访问的云扉 API | 返回给面板的数据 |
| --- | --- | --- |
| `POST /aigate/account` | `POST /user/balance` | `{ ok, balance, updatedAt }` |
| `POST /aigate/create-options` | `GET /instance/skuList` | 过滤后的可用 SKU、`priceLabel`、刷新时间 |
| `POST /aigate/create-instance` | `POST /instance/start` | 标准化的实例摘要和创建状态 |

`/aigate/instances` 的结果必须保留足以区分创建中、运行中、已停止和已释放的状态，用于严格控制创建卡的显示。开发预览的 mock API 镜像上述端点和状态转移。

## 预设镜像和可配置项

云扉创建实例要求 `areaName`、`imageId` 和 `imageType`。本项目不能把特定账户或区域的镜像 ID 硬编码到仓库，因此 bridge 从忽略 Git 的本地配置读取一个“预设 ComfyUI 镜像”配置：

```json
{
  "aigateCreate": {
    "areaName": "…",
    "imageId": "…",
    "imageType": "2"
  }
}
```

若本地桥未配置这些值，创建卡显示“本机尚未配置预设 ComfyUI 镜像”，不发送创建请求，并给出简明的本机配置指引。该策略避免误用错误区域或用户私有镜像。

## 错误、刷新和安全

- 余额、SKU 和实例列表独立刷新；余额刷新不重载实例，实例轮询不重复请求 SKU。
- 初次进入云扉设置、点击刷新、创建提交后和创建中轮询都会刷新实例状态；创建中按既有 10 秒实例刷新节奏观察状态。
- 创建请求进行中锁定“确认创建”按钮；网络或 API 失败后解除锁定并保留用户所选 SKU。
- 价格显示以 bridge 当前返回的 `priceLabel` 为准；创建确认使用同一 SKU，但最终计费以云扉创建接口为准。
- Token 不从 bridge 响应回传；任何异常、日志和测试 fixture 均不包含真实 Token。

## 测试与验收

- Python 单元测试：余额、SKU、创建请求的成功、认证失败、超时和 API 错误；创建配置缺失；Token 脱敏。
- Dev mock 测试：空列表显示创建入口；提交创建后进入创建中；存在任何实例时入口隐藏；余额和价格刷新。
- 浏览器/UXP 预览：选择、确认、创建中和错误状态的交互；窄面板无横向溢出。
- 回归：既有云扉实例发现、启动、关闭、释放和面板隐藏时关闭受管实例的逻辑不变。

## 非目标

- 不在插件中提供区域、镜像、GPU 数量或云扉账户充值的完整管理控制台；
- 不创建第二台实例或提供实例并行调度；
- 不在前端写死价格、镜像 ID 或账户专属数据；
- 不把云扉 Token 发送到原生 ComfyUI 工作流请求。
