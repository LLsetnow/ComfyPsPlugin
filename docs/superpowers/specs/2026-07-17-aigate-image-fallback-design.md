# 云扉默认镜像回退设计

## 目标

创建云扉 ComfyUI 实例时，默认加载名为 `comfyui-boogu-edit-int8-20260716` 的镜像。系统优先使用当前账号的个人镜像；个人镜像不存在或不兼容所选 GPU 时，回退到同名社区镜像。

## 本机配置

`bridge/config.json` 中的 `aigateCreate` 支持以下命名镜像配置：

```json
"aigateCreate": {
  "areaName": "华东一区",
  "imageName": "comfyui-boogu-edit-int8-20260716",
  "imageTypes": ["3", "2"]
}
```

- `"3"` 是个人镜像，`"2"` 是社区镜像；数组顺序定义优先级。
- 保留既有数值 `imageId` + `imageType` 配置作为显式覆盖路径，兼容已部署配置。
- 默认镜像名称由 bridge 定义；本机配置省略 `imageName` 时使用该默认值。

## 解析与创建流程

1. 面板先读取实例列表并只在控制台确认为空时显示创建卡。
2. 用户选择 SKU 并确认创建后，bridge 在创建锁内解析镜像：
   - 若已配置有效数值 `imageId`，直接使用它和配置的 `imageType`；
   - 否则按 `imageTypes` 顺序查询镜像列表，要求镜像名称精确相等；
   - 个人镜像查询属于当前 Token 的镜像分页结果；社区镜像查询同时带入区域、SKU 和镜像名。
3. bridge 将解析出的 `worksId` 和对应 `imageType` 传给现有实例创建请求。
4. 两类镜像均未找到时返回 `409 AIGATE_IMAGE_NOT_FOUND`，明确提示已尝试个人和社区镜像，绝不创建实例。

## 边界与安全

- 镜像解析在 bridge 内完成，面板不会得到镜像列表、Token 或数值镜像 ID。
- 镜像选择在服务端创建锁内执行，保留“已有任意实例不得再创建”的保护。
- GPU 规格查询仅依赖区域，不因尚未解析镜像而失败。
- 个人镜像和社区镜像都必须精确匹配默认名称；不允许名称模糊匹配或静默换成其他镜像。

## 验证

单元测试覆盖：个人镜像命中、个人缺失后社区命中、两者均缺失的安全 409、显式 `imageId` 的向后兼容、社区查询携带区域和 SKU。README 与示例配置说明默认名称、优先级及显式 ID 覆盖。
