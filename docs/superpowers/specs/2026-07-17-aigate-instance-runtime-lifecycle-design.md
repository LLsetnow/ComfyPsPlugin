# 云扉实例运行时间与生命周期控制设计

## 目标

在云扉 ComfyUI 设置页显示由插件启动实例的本地运行时间，提供启动、关闭和释放操作，并在插件、Photoshop 或本地桥正常退出时尽力关闭由插件管理的运行实例。

## 非目标

- 不估算或展示云扉费用。
- 不自动释放实例；退出清理只执行关闭。
- 不保存云扉 Token 到 bridge 配置文件、磁盘或日志。
- 不承诺强制退出、崩溃或断电后的关闭成功。

## 本地运行时间

插件在 `localStorage` 的 `comfyps.aigateInstanceLifecycle.v1` 保存按实例 ID 索引的记录：

```json
{
  "1130287682570289152": {
    "managed": true,
    "pendingStart": false,
    "startedAt": 1784219400000
  }
}
```

`open` 请求成功后，插件仅写入 `managed: true` 和 `pendingStart: true`。随后刷新实例列表；第一次观察到该实例 `operationStatus === "2"` 时，写入本机 `Date.now()` 到 `startedAt`，清除 `pendingStart`。这使显示的运行时间不包含镜像拉取或开机等待。

运行中实例显示 `运行 HH:MM:SS`。设置页打开且存在待启动或已开始计时的受管实例时，插件每 10 秒刷新实例列表，并每秒重新渲染本地时间。实例已运行但没有本地 `startedAt` 时显示“开始时间未知”，不得推测开始时间。关闭成功后保留本地记录但不显示运行计时；释放成功后删除记录。

## 设置页交互

实例列表沿用现有紧凑行：名称、ID、云扉状态与 ComfyUI 可用性在左侧，运行时间位于状态之后，操作位于右侧。

| 云扉状态 | 显示与操作 |
| --- | --- |
| `2` 运行中 | 显示本地运行时间；显示“关闭”和红色描边“释放” |
| `7` 已关机、`22` 已关机且镜像已提交 | 显示“已停止”；显示“启动”和红色描边“释放” |
| `4` 已释放 | 显示“已释放”；不显示操作 |
| 其他过渡状态 | 显示状态文字；禁用操作，直到手动或定时刷新得到稳定状态 |

“释放”须二次确认，文案说明该操作会释放云扉实例且不能用“启动”恢复；确认后调用云扉 `instance/release`。关闭只调用 `instance/close`，保留实例供后续启动。

## 桥接 API 与状态

现有 `POST /aigate/instances` 请求增加可选 `managedInstanceIds` 数组。桥在内存中维护 `instanceId -> Token` 的受管映射；仅接收当前请求携带的 Token，不写入日志或文件。插件在刷新时同步其本地 `managed` 实例，使桥重启后能在下一次刷新恢复退出清理能力。

`POST /aigate/instance-action` 的 `action` 扩展为 `open`、`close`、`release`：

- `open` 成功后在桥内存注册该实例；插件写入待启动本地记录。
- `close` 不删除受管记录。
- `release` 成功后从桥内存与插件本地记录删除该实例。

新增 `POST /aigate/close-managed`，参数为当前 Token 与插件受管实例 ID 列表。桥验证 ID、合并内存映射，并通过 `asyncio.gather` 并发调用 `instance/close`；每个失败只返回安全错误摘要，不中止其余关闭。该端点永不调用 `release`。

桥的 `on_shutdown` 钩子对内存中的受管实例执行相同的并发关闭。桥重启也属于正常退出，因此会触发关闭。没有 Token 映射、进程被强制终止或网络不可用时，只记录安全错误并结束退出流程。

## UXP 生命周期

面板监听 Photoshop UXP 的 `uxpcommand` 隐藏事件，并以 `beforeunload` 作为补充。在正常生命周期事件中，插件尝试把当前受管实例提交至 `/aigate/close-managed`；优先使用 `navigator.sendBeacon`，不可用时发出短超时请求。请求不等待 UI 完成，不阻塞 Photoshop 退出。

此机制仅保证最佳努力：强制关闭 Photoshop、系统崩溃或断电可能阻止面板与桥执行清理。桥的正常 shutdown 钩子是第二道兜底。

## 错误处理与安全

- 所有本地桥响应均为 JSON；面板在解析前检查 HTTP 状态并展示安全错误文本。
- 云扉 Token 不包含在 UI 错误、日志、实例摘要或测试断言中。
- 释放确认、关闭清理和运行时间均只作用于本地标记为 `managed` 的实例。
- 原生 ComfyUI 的上传、提交、轮询和下载路径保持无 Authorization header。

## 验证

- Python 适配器测试覆盖 `release`、受管实例并发关闭、未授权/无效 action，以及不泄露 Token。
- 桥端点测试覆盖实例列表同步、释放、关闭清理与安全 JSON 错误。
- 开发 mock 测试覆盖三种操作与清理端点。
- 面板 Node 测试覆盖首次观察运行时写入起点、重载后继续计时、未知开始时间、稳定状态操作，以及释放后的本地记录删除。
- 手工验证：通过设置页启动一个实例，确认首次运行状态出现后开始计时；隐藏面板或正常退出 Photoshop 后确认实例进入关闭流程；释放操作必须经过确认并从列表消失。
